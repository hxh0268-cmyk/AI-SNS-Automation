## 0. この設計書について

### 0.1 目的

v1.3 品質パイプラインにおいて **TEXT rootCause** の改善対象は `smart_auto_fix` にルーティングされます。**v1.4.0 では実接続完了** — Smart Auto Fix → Regeneration Engine → Nano Banana adapter → Gemini ReReview → scoreSummary まで quality loop に接続されています。

- TEXT rootCause を **Smart Auto Fix（SAF）** で実改善する
- 改善後に **Regeneration Engine** 経由で画像を再生成する
- 既存の **Gemini ReReview → scoreSummary 更新 → quality loop 継続** に接続する

### 0.2 設計方針（v1.4 追加）

| 方針 | 内容 |
|------|------|
| **責務分離** | SAF は TEXT 向けテキスト・プロンプト改善のみ。Nano Banana は画像再生成・画像改善のみ |
| **直接呼び出し禁止** | SAF から Nano Banana を **直接 import / 呼び出ししない** |
| **Regeneration Engine** | SAF と画像再生成の間に **共通 IF** を挟む |
| **adapter 差し替え** | v1.4 は内部 adapter として Nano Banana。v1.5+ で OpenAI regenerate adapter に差し替え可能 |
| **SAF の安定性** | 将来別画像モデルを追加しても **SAF 側は変更不要** |
| dry-run 標準 | デフォルト API 未呼び出し（v1.3 継承） |
| 非破壊 | `run_daily.sh` は変更しない |
| スコープ外 | `openai_regenerate` 実装、`--resume`、GitHub Actions |

### 0.3 v1.3 設計からの変更点

| 項目 | v1.3 設計 | v1.4 設計（修正後） |
|------|-----------|---------------------|
| TEXT 改善 | SAF + OpenAI 再生成（直結想定） | SAF → **Regeneration Engine** → adapter |
| Nano Banana 役割 | LAYOUT/STYLE + TEXT 暫定ブリッジ（議論段階） | **adapter 実装の1つ**。SAF からは見えない |
| 新規 lib | なし | `regeneration_engine.js` + adapter(s) |

## 1. 変更後の処理フロー

### 1.1 TEXT rootCause — apply モード（基本フロー）

```
TEXT rootCause（scoreSummary / classifyRootCause）
        ↓
Smart Auto Fix（slide.md / prompt.md 追記・backup）
        ↓
Regeneration Engine（共通 IF：入力を adapter に委譲）
        ↓
Nano Banana adapter（v1.4 暫定実装）
        ↓
output/carousel/improved/slideXX.png + manifest item
        ↓
Gemini ReReview（applyReReviewFromManifest）
        ↓
mergeReviewResultIntoScoreSummary
        ↓
shouldContinueImprovement → 次ラウンド or stopReason
```

### 1.2 レイヤ責務（呼び出し方向）

```
┌──────────────────────────────────────────────────────────────┐
│  pipeline_improvement.js（オーケストレーション）               │
│    processImprovementTarget(TEXT)                             │
│      1. smart_auto_fix.js  … applyFixForSlide()              │
│      2. regeneration_engine.js … regenerateImage()           │
│         └─ adapter: nano_banana_adapter.js（v1.4）           │
└──────────────────────────────────────────────────────────────┘

※ SAF は regeneration_engine を知らない
※ regeneration_engine は SAF を知らない
※ 両者の接続は pipeline_improvement が担う
```

### 1.3 LAYOUT / STYLE / BOOST — 既存フロー（v1.4 変更なし）

```
LAYOUT / STYLE / BOOST（80–89点）
        ↓
processNanoBananaTarget（既存）
        ↓
Gemini ReReview
        ↓
scoreSummary 更新（source: nano_banana_re_review）
        ↓
quality loop 継続
```

Nano Banana **直接ルート** と **Regeneration Engine 経由ルート** は pipeline 内で共存します。TEXT のみ SAF + Regeneration Engine チェーンを使います。

### 1.4 dry-run モード

| 段階 | 動作 |
|------|------|
| Smart Auto Fix | **計画のみ** — 修正予定ファイル・rootCause・改善方針をログ / report に出力。ファイル変更なし |
| Regeneration Engine | **planned** — adapter 名（`nano_banana`）と入力パスを表示。API 未呼び出し |
| Gemini ReReview | **planned**（v1.3 既存どおり） |
| manifest | dry-run 用 item を書く場合は `status: "planned"` / `dryRun: true`（v1.3 慣行に合わせる） |

dry-run 標準は維持します。`--apply` 指定時のみ SAF 追記・Regeneration Engine API・Gemini ReReview が実行されます。

### 1.5 quality loop 内の位置（変更なし）

`quality_pipeline.js` の `runImprovementLoop()` 構造は **維持** します。

```
while (round < maxRounds):
  IMPROVEMENT  … tool 別 dispatch（TEXT → SAF + Regen Engine）
  RE_REVIEW    … manifest ベース Gemini
  shouldContinueImprovement()
```

新 Phase の追加は行いません。TEXT チェーンは **IMPROVEMENT Phase 内** で完結します。

### 1.6 シーケンス図（TEXT / apply）

```mermaid
sequenceDiagram
  participant Loop as runImprovementLoop
  participant IMP as pipeline_improvement
  participant SAF as smart_auto_fix lib
  participant RE as regeneration_engine
  participant NB as nano_banana adapter
  participant Man as manifest.json
  participant RR as applyReReviewFromManifest
  participant SS as scoreSummary

  Loop->>IMP: applyImprovementPlan (TEXT target)
  IMP->>SAF: applyFixForSlide(slideId, rootCause)
  SAF-->>IMP: SmartAutoFixResult
  IMP->>RE: regenerateImage(RegenerationRequest)
  RE->>NB: regenerate(request)
  NB-->>RE: RegenerationResult
  RE-->>IMP: RegenerationResult
  IMP->>Man: manifest item (improved|failed)
  Loop->>RR: read manifest
  RR->>RR: Gemini review PNG
  RR->>SS: mergeReviewResultIntoScoreSummary
  Loop->>Loop: shouldContinueImprovement
```

## 3. `regeneration_engine.js` の責務

### 3.1 役割

**改善済みテキスト / 改善済み prompt を入力として、画像を再生成する共通インターフェース** です。

- Smart Auto Fix・Nano Banana 直接ルート・将来の OpenAI adapter **いずれからも** 同じ API で呼べる
- rootCause の解釈は **行わない**（呼び出し元が入力を整える）
- v1.4 では **Nano Banana adapter のみ登録**。内部実装の詳細は adapter に閉じる

### 3.2 公開 API（案）

```javascript
/**
 * @typedef {object} RegenerationRequest
 * @property {string} slideId
 * @property {string} sourceImagePath      … 元 PNG（プロジェクト相対）
 * @property {string} outputPath           … 出力 PNG（プロジェクト相対）
 * @property {string} promptPath          … 改善済み prompt md
 * @property {string[]} [changedTextPaths] … SAF が変更した md 一覧（監査用）
 * @property {string} [adapterId]         … 未指定時は config デフォルト
 * @property {boolean} dryRun
 * @property {object} [config]            … timeout / retry / model 等
 */

/**
 * @typedef {object} RegenerationResult
 * @property {string} slideId
 * @property {string} adapterId            … 実際に使用した adapter
 * @property {"improved"|"failed"|"planned"|"skipped"} status
 * @property {string} outputPath
 * @property {string | null} error
 * @property {number} elapsedMs
 * @property {number} attempts
 */

/** adapter 登録 */
export function registerRegenerationAdapter(adapterId, adapter);

/** 画像再生成（adapter に委譲） */
export async function regenerateImage(request, options);

/** dry-run 計画（API 未呼び出し） */
export function planRegeneration(request, options);

/** 利用可能 adapter 一覧 */
export function listRegenerationAdapters();
```

### 3.3 adapter インターフェース（案）

```javascript
/**
 * @typedef {object} RegenerationAdapter
 * @property {string} id                   … "nano_banana" | "openai"（将来）
 * @property {string} label
 * @property {(request: RegenerationRequest, context: object) => Promise<RegenerationResult>} regenerate
 * @property {(request: RegenerationRequest) => object} [plan]  … dry-run 用
 */
```

### 3.4 v1.4 デフォルト adapter 選択

| rootCause | IMPROVEMENT tool | Regeneration adapter（v1.4） |
|-----------|------------------|------------------------------|
| TEXT | `smart_auto_fix` | `nano_banana`（暫定） |
| LAYOUT / STYLE / BOOST | `nano_banana` | **Regeneration Engine を経由しない**（既存 `processNanoBananaTarget`） |
| PROMPT | `openai_regenerate` | placeholder（v1.4 未実装） |

**設計意図:** LAYOUT/STYLE は「画像改善」が主目的のため、既存 Nano Banana 直呼びを維持。TEXT は「テキスト修正 → 再生成」が主目的のため Regeneration Engine 経由。

v1.5 で OpenAI adapter を追加した場合、**pipeline_improvement の adapter 選択ロジックのみ変更**し、SAF は変更しません。

### 3.5 Regeneration Engine がやらないこと

| 項目 | 担当 |
|------|------|
| slide.md / prompt.md 追記 | Smart Auto Fix |
| rootCause 分類 | `root_cause.js` / `pipeline_improvement.js` |
| Gemini 再レビュー | `applyReReviewFromManifest` |
| scoreSummary 集計 | `pipeline_score.js` |
| quality loop 制御 | `quality_pipeline.js` |

## 4. Smart Auto Fix と Regeneration Engine の入出力

### 4.1 Smart Auto Fix

#### 入力（`applyFixForSlide` / `planFixForSlide`）

| フィールド | 型 | 説明 |
|------------|-----|------|
| `slideId` | `string` | 例: `slide02` |
| `rootCause` | `string` | v1.4 では **`TEXT` のみ** 実接続（他は呼ばれない想定） |
| `score` | `number` | 改善前 score |
| `issues` | `string[]` | scoreSummary slide から |
| `recommendations` | `string[]` | 同上 |
| `dryRun` | `boolean` | true = 計画のみ |
| `config` | `object` | backup 先等（任意） |

#### 出力（`SmartAutoFixResult`）

| フィールド | 型 | 説明 |
|------------|-----|------|
| `slideId` | `string` | |
| `rootCause` | `string` | |
| `status` | `string` | `planned` / `applied` / `skipped` / `failed` |
| `changedFiles` | `string[]` | 追記した md（apply 時） |
| `backedUpFiles` | `string[]` | backup パス |
| `skippedFiles` | `string[]` | 既に SAF 指示あり等 |
| `promptPath` | `string` | 改善対象 prompt md（Regeneration 入力用） |
| `slideMdPath` | `string \| null` | TEXT 時の slide md |
| `error` | `string \| null` | |
| `fixPlan` | `object` | dry-run 用 strategy 一覧 |

**SAF 出力に含めないもの:** `outputPath`（PNG）、API metrics — これらは Regeneration Engine の責務。

### 4.2 Regeneration Engine

#### 入力（`RegenerationRequest` — pipeline が SAF 結果から組み立て）

| フィールド | 来源 |
|------------|------|
| `slideId` | target |
| `promptPath` | `SmartAutoFixResult.promptPath` |
| `changedTextPaths` | `SmartAutoFixResult.changedFiles` |
| `sourceImagePath` | `images/carousel/output/slideXX.png`（固定規約） |
| `outputPath` | `output/carousel/improved/slideXX.png` |
| `adapterId` | config デフォルト `"nano_banana"`（v1.4） |
| `dryRun` | pipeline config |

#### 出力（`RegenerationResult`）

| フィールド | 説明 |
|------------|------|
| `adapterId` | 使用 adapter（監査・manifest 用） |
| `status` | `improved` / `failed` / `planned` |
| `outputPath` | 生成 PNG パス |
| `elapsedMs` / `attempts` | metrics 用 |
| `error` | 失敗理由 |

### 4.3 pipeline_improvement による接続（擬似コード）

```javascript
async function processSmartAutoFixTarget(target, slide, context) {
  const safResult = context.dryRun
    ? planFixForSlide({ slideId: target.slideId, rootCause: "TEXT", ... })
    : await applyFixForSlide({ ... });

  if (safResult.status === "failed") {
    return buildFailedTargetResult(target, safResult);
  }

  const regenResult = context.dryRun
    ? planRegeneration(buildRegenerationRequest(safResult, target))
    : await regenerateImage(buildRegenerationRequest(safResult, target), { metrics: context.metrics });

  return buildCombinedManifestItem(target, safResult, regenResult);
}
```

**要点:** SAF と Regeneration Engine の間のデータ変換は **pipeline_improvement のみ** が担い、相互 import は禁止。

## 6. scoreSummary source の扱い

### 6.1 source 値一覧（v1.4）

| source | 設定タイミング | 経路 |
|--------|----------------|------|
| `image_review` | IMAGE_REVIEW | 初回読込 |
| `nano_banana_re_review` | RE_REVIEW 後 | LAYOUT/STYLE/BOOST — Nano Banana 直呼び |
| **`smart_auto_fix_re_review`** | RE_REVIEW 後 | TEXT — SAF → Regeneration Engine |
| `unknown` | フォールバック | |

### 6.2 merge ルール

`mergeReviewResultIntoScoreSummary()` を以下の方針で一般化:

1. manifest item の `improvementPipeline` を参照
2. `"smart_auto_fix"` を含む → `source: "smart_auto_fix_re_review"`
3. `"nano_banana"` のみ / tool が `nano_banana` → `source: "nano_banana_re_review"`
4. 再レビュー成功 item のみ score / rootCause を上書き（v1.3 既存）

```javascript
function resolveReviewSource(manifestItem) {
  const pipeline = manifestItem.improvementPipeline ?? [];
  if (pipeline.includes("smart_auto_fix")) {
    return "smart_auto_fix_re_review";
  }
  if (manifestItem.tool === "nano_banana" || pipeline.includes("nano_banana")) {
    return "nano_banana_re_review";
  }
  return "unknown";
}
```

### 6.3 export / report への影響

| モジュール | 対応 |
|------------|------|
| `pipeline_export.js` | improved 採用条件に `smart_auto_fix_re_review` を許容 |
| `pipeline_report.js` | `improvementTool` / `source` 表示、Next Actions から「v1.4 以降予定」文言を削除 |
| `pipeline_state.json` | scoreSummary.slides[].source に反映 |

### 6.4 rootCause の materialize（推奨・v1.4）

IMAGE_REVIEW 後、または IMPROVEMENT plan 作成時に `scoreSummary.slides[].rootCause` を **明示保存** することを推奨（report / デバッグ用）。classify ロジック自体は `root_cause.js` を共用。
